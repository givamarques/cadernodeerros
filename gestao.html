Resumo de Gerenciamento de Serviços de TI e Gerência de Projetos

1. Gerenciamento de Serviços de TI

1.1 Biblioteca ITIL® versão 4: Suporte a Serviços e Entrega de Serviços

A ITIL® (Information Technology Infrastructure Library) é uma biblioteca de boas práticas para gerenciamento de serviços de TI, com foco em alinhar os serviços de TI às necessidades do negócio. A versão 4 inclui práticas modernas de suporte e entrega de serviços, proporcionando maior agilidade e flexibilidade.

1.2 COBIT 2019

COBIT (Control Objectives for Information and related Technology) é um framework para governança e gerenciamento de TI que ajuda as organizações a alcançar seus objetivos estratégicos. A versão 2019 atualiza e torna mais flexível a abordagem do COBIT, permitindo a adaptação às necessidades da empresa.

2. Infraestrutura de Soluções

2.1 Conhecimentos de Ambiente Microsoft

  • Active Directory: Serviço da Microsoft que permite gerenciar usuários e recursos em uma rede, fornecendo autenticação e autorização centralizadas.
  • Exchange: Plataforma de mensagens que oferece serviços de e-mail, calendário e contatos para empresas.
  • Windows Server: Sistema operacional da Microsoft destinado ao uso em servidores, oferecendo serviços de rede, armazenamento e gerenciamento de usuários.

2.2 Ferramenta MQ Series

O IBM MQ Series é um middleware de mensageria que permite a troca de mensagens entre diferentes aplicações, garantindo a entrega confiável e segura de dados.

2.3 Noções de Protocolos

  • FTP (File Transfer Protocol): Protocolo usado para transferência de arquivos pela internet.
  • NetBIOS (Network Basic Input/Output System): Protocolo que permite a comunicação entre computadores em uma rede local.

2.4 CMDB

O CMDB (Configuration Management Database) é um banco de dados utilizado para armazenar informações sobre todos os componentes de TI, seus relacionamentos e configurações, facilitando o gerenciamento dos ativos de TI.

2.5 Armazenamento e Storages

Sistemas de armazenamento e storages são usados para armazenar dados de forma segura e confiável, seja em dispositivos físicos ou em nuvem, como cloud storage.

2.6 Cloud Computing e Cloud Storage

Cloud Computing refere-se ao uso de recursos de computação (servidores, armazenamento) em servidores remotos acessíveis pela internet. Cloud Storage é uma forma de armazenar dados remotamente na nuvem, proporcionando escalabilidade e disponibilidade.

2.7 Virtualização de Servidores

A virtualização de servidores é uma tecnologia que permite criar várias máquinas virtuais em um único servidor físico, otimizando o uso dos recursos de hardware e simplificando o gerenciamento.

2.8 Sistemas Operacionais Windows e Linux

Conhecimentos sobre Windows e Linux são fundamentais para administração e gerenciamento de servidores, incluindo tarefas como instalação, configuração e solução de problemas.

2.9 Conhecimentos de Backup (Full, Restore, Incremental)

  • Full Backup: Cópia completa de todos os dados.
  • Restore: Processo de recuperar dados de um backup.
  • Backup Incremental: Cópia dos dados que foram alterados desde o último backup realizado.

2.10 Gerenciamento e Configuração de Servidores de Aplicação

  • Apache: Servidor web de código aberto, amplamente utilizado para hospedar sites e aplicações.
  • Tomcat: Servidor de aplicações Java que executa Servlets e JSPs.
  • JBoss: Servidor de aplicações Java EE utilizado para desenvolver e executar aplicações corporativas.
  • IIS (Internet Information Services): Servidor web da Microsoft utilizado para hospedar sites e aplicações web no Windows.

2.11 Red Hat OpenShift

Red Hat OpenShift é uma plataforma de Kubernetes que facilita a criação, implementação e gerenciamento de aplicações em containers, oferecendo recursos para escalabilidade e automação.

3. Gerência de Projetos

3.1 Processos Ágeis

Processos ágeis são metodologias de desenvolvimento que focam em entregas rápidas e adaptáveis, promovendo colaboração constante com o cliente e flexibilidade para mudanças.

3.2 SCRUM

SCRUM é uma metodologia ágil que utiliza ciclos curtos de desenvolvimento chamados de sprints, permitindo a entrega contínua de funcionalidades e a adaptação rápida às mudanças.

3.3 SAFe

SAFe (Scaled Agile Framework) é um framework que aplica princípios ágeis em nível corporativo, permitindo que grandes equipes trabalhem de forma colaborativa e coordenada.

3.4 Kanban

Kanban é uma metodologia ágil que utiliza um quadro visual para gerenciar o fluxo de trabalho, focando na melhoria contínua e na eficiência do processo.

3.5 Metodologias, Técnicas e Ferramentas da Gerência de Projetos

Envolve o uso de metodologias como PMBOK, técnicas como o diagrama de Gantt, e ferramentas como Microsoft Project e Trello para planejar, monitorar e executar projetos de forma eficiente.
