Resumo sobre Banco de Dados

1. Conceitos e Fundamentos

Os conceitos e fundamentos de bancos de dados envolvem a compreensão de como organizar, armazenar e gerenciar dados para torná-los facilmente acessíveis e seguros. Envolve terminologia básica como tabelas, registros, chaves e relacionamentos.

2. Noções de Administração dos SGBDs Oracle, PostgreSQL, SQLServer e MySQL

A administração dos SGBDs (Sistemas de Gerenciamento de Banco de Dados) Oracle, PostgreSQL, SQLServer e MySQL envolve a configuração, manutenção, otimização e gerenciamento da segurança e desempenho desses sistemas. Cada um tem características específicas, mas todos são amplamente usados para armazenar e processar dados corporativos.

3. Linguagem SQL e PL/SQL: Consultas e Subconsultas

SQL (Structured Query Language) é usada para consultar e manipular dados em um banco de dados relacional. PL/SQL é uma extensão da SQL usada no Oracle, permitindo a criação de scripts mais complexos, como funções e blocos de código reutilizáveis.

4. Gatilho (Trigger), Visão (View), Function e Stored Procedures, Packages

  • Gatilho (Trigger): Um procedimento que é executado automaticamente em resposta a certos eventos no banco de dados.
  • Visão (View): Uma consulta salva que age como uma tabela virtual para facilitar o acesso a informações específicas.
  • Function e Stored Procedures: Blocos de código armazenados que executam tarefas específicas no banco de dados.
  • Packages: Conjuntos agrupados de funções e procedimentos, geralmente usados para organizar código no Oracle.

5. Tratamento de Erros

O tratamento de erros é o processo de captura e gerenciamento de erros que podem ocorrer durante a execução de uma consulta ou procedimento, garantindo que o sistema lide com falhas de forma previsível.

6. Cursores

Cursores são utilizados para iterar sobre linhas retornadas por uma consulta. Eles são especialmente úteis em PL/SQL para manipular dados linha a linha.

7. Array

Arrays são estruturas de dados que armazenam coleções de valores, úteis para manipulação de conjuntos de informações em PL/SQL ou outras linguagens associadas ao banco de dados.

8. Projeto e Modelagem de Banco de Dados Relacional

O projeto e modelagem de banco de dados envolvem a criação de um esquema que define como os dados serão armazenados, incluindo tabelas, relacionamentos e restrições.

9. Modelo Entidade Relacionamento

É uma representação gráfica que descreve os dados e seus relacionamentos, sendo uma ferramenta fundamental para o planejamento e design de bancos de dados relacionais.

Entidade = retangulo / Relacionamento = Losango (podem aparecer juntos, formando a bandeira do BR)

10. Normalização

Processo de organização de dados para minimizar a redundância e melhorar a integridade. Envolve dividir grandes tabelas em tabelas menores relacionadas.

11. Administração de Dados

Refere-se à gestão dos dados de uma organização, incluindo definição, criação, segurança e política de uso dos dados.

12. Conceitos de Data Warehouse, Data Mining, OLAP

  • Data Warehouse: Repositório centralizado para dados históricos, usados para relatórios e análise.

    • Data Warehouse

    O que é?

    Data Warehouse é um sistema utilizado para armazenar grandes volumes de dados, organizando-os de forma a facilitar a consulta e análise. Ele centraliza informações de diversas fontes, integrando os dados para fins analíticos e de suporte à decisão.

    Para que serve?

    O Data Warehouse serve para consolidar dados históricos de diferentes sistemas, possibilitando uma visão ampla dos processos empresariais. Ele é amplamente utilizado para relatórios, análise de tendências, inteligência de negócios (BI) e suporte à tomada de decisões estratégicas.

    Como funciona?

    O Data Warehouse funciona coletando e integrando dados de várias fontes, como bancos de dados operacionais, planilhas e outras fontes de dados. Esses dados são então transformados e carregados em um banco de dados central, onde são organizados de forma a facilitar a análise. O processo pode ser resumido em três etapas principais:

    • Extração: coleta de dados de diferentes fontes.
    • Transformação: limpeza e padronização dos dados para torná-los consistentes.
    • Carga: armazenamento dos dados no Data Warehouse.

    Depois de armazenados, os dados podem ser consultados através de ferramentas de Business Intelligence (BI), que permitem a criação de relatórios, dashboards e outras formas de visualização para análise estratégica.

  • Data Mining: Processo de análise de grandes volumes de dados para descobrir padrões ocultos.

    • Data Mining

    O que é?

    Data Mining, ou Mineração de Dados, é o processo de explorar grandes volumes de dados para identificar padrões, tendências e relações úteis. Ele envolve a aplicação de técnicas estatísticas, de aprendizado de máquina e algoritmos para extrair conhecimento significativo a partir dos dados.

    Para que serve?

    Data Mining serve para descobrir informações ocultas nos dados, ajudando as empresas a tomar decisões mais informadas. Ele é utilizado em áreas como marketing, finanças, medicina e segurança, permitindo prever comportamentos, identificar fraudes, segmentar clientes e melhorar processos.

    Como funciona?

    O processo de Data Mining envolve várias etapas:

    • Preparação dos dados: inclui a coleta, limpeza e organização dos dados para garantir que estejam prontos para análise.
    • Exploração dos dados: aplicação de técnicas estatísticas e algoritmos de aprendizado de máquina para identificar padrões e relações significativas.
    • Interpretação e validação: análise dos resultados para garantir que os padrões encontrados sejam precisos e úteis para o contexto do problema.

    Data Mining utiliza técnicas como classificação (para categorizar dados), agrupamento (para encontrar grupos de itens semelhantes), e regras de associação (para identificar relações entre variáveis). É uma ferramenta poderosa para transformar dados em informações estratégicas e obter vantagens competitivas.

  • OLAP (Online Analytical Processing): Tecnologia que permite realizar consultas analíticas complexas sobre dados armazenados.

    • OLAP (Online Analytical Processing)

    OLAP é uma tecnologia utilizada para realizar consultas analíticas em grandes volumes de dados. É especialmente útil em ambientes de Data Warehousing, onde os dados são organizados para suportar análises e tomadas de decisão.

    Para que serve?

    OLAP serve para fornecer uma maneira rápida e eficiente de acessar e analisar dados multidimensionais. Ele permite que usuários empresariais explorem informações de diferentes perspectivas (como tempo, produto, região) para responder a perguntas complexas e descobrir padrões nos dados.

    Como funciona?

    OLAP utiliza estruturas chamadas "cubos OLAP", que organizam os dados de forma multidimensional, permitindo consultas rápidas. Esses cubos são formados por dimensões (ex.: tempo, produto) e medidas (ex.: vendas, receita). As operações básicas incluem:

    • Drill-down: explorar dados em maior detalhe.
    • Roll-up: resumir dados para um nível mais alto.
    • Slice and Dice: filtrar e reorganizar os dados para diferentes visões.

    Com OLAP, é possível realizar análises ad-hoc, que não são limitadas por consultas pré-definidas, sendo uma ferramenta poderosa para suporte à tomada de decisão em empresas.

13. Tuning em Banco de Dados

Tuning envolve otimizar o desempenho de consultas e operações no banco de dados para melhorar a eficiência e o tempo de resposta.

14. Segurança de Banco de Dados

A segurança de banco de dados abrange técnicas e práticas para proteger dados contra acessos não autorizados, perda e corrupção, garantindo integridade e confidencialidade.

15. NoSQL: Conceitos Básicos, Tipos de Bancos

NoSQL refere-se a bancos de dados não relacionais, que são escaláveis e flexíveis. Existem diferentes tipos, como:

  • Bancos Orientados a Grafos: Usam grafos para representar relacionamentos complexos.
  • Bancos de Colunas: Armazenam dados em colunas, como o Cassandra.
  • Chave/Valor: Estruturas simples de chave e valor, como o Redis.
  • Documentos: Armazenam dados como documentos, geralmente JSON, como o MongoDB.

16. MongoDB

Um banco de dados NoSQL que armazena dados em formato de documento JSON, proporcionando flexibilidade e escalabilidade.

17. Big Data: Fundamentos

Big Data refere-se a grandes volumes de dados complexos e variados que requerem processamento especializado. Envolve técnicas e ferramentas para coletar, armazenar e analisar dados que não cabem em bancos de dados tradicionais.

18. Técnicas de Preparação e Apresentação de Dados

Essas técnicas incluem a limpeza, transformação e visualização dos dados para torná-los mais compreensíveis e úteis para análise e tomada de decisão.
