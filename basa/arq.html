Resumo de Arquitetura e Desenvolvimento de Software

1. Service Oriented Architecture (SOA)

A Service Oriented Architecture (SOA) é uma abordagem de arquitetura que organiza sistemas em forma de serviços reutilizáveis e independentes. Serve para integrar diferentes sistemas, facilitando a interoperabilidade e a escalabilidade.

2. SOAP

SOAP (Simple Object Access Protocol) é um protocolo de comunicação usado em serviços web para troca de informações estruturadas em formato XML, garantindo a interoperabilidade entre diferentes plataformas.

3. REST

REST (Representational State Transfer) é um estilo arquitetural para a criação de serviços web que utiliza HTTP como protocolo. Ele facilita a comunicação entre sistemas através de operações como GET, POST, PUT e DELETE.

4. Design Patterns

Design Patterns são soluções reutilizáveis para problemas comuns de design de software. Eles ajudam a garantir boas práticas e facilitam a manutenção e evolução dos sistemas.

5. Padrão Model-View-Controller (MVC)

O padrão MVC (Model-View-Controller) é uma arquitetura que separa a lógica de negócios (Model), a interface de usuário (View) e o controle de fluxo da aplicação (Controller), promovendo a modularidade e facilitando a manutenção.

6. REST, JSON, ANGULARJS, Microserviços

REST e JSON são frequentemente usados juntos para construir APIs que trocam informações de forma eficiente. AngularJS é um framework JavaScript usado para criar interfaces de usuário dinâmicas. Microserviços são uma abordagem de desenvolvimento de sistemas onde as funcionalidades são divididas em pequenos serviços independentes que se comunicam entre si.

7. Fundamentos de DevOps: CI/CD (Continuous Integration/Continuous Delivery)

DevOps é uma cultura que integra equipes de desenvolvimento e operações para melhorar a entrega de software. CI/CD refere-se a práticas de Continuous Integration (integração contínua) e Continuous Delivery (entrega contínua), automatizando o teste e a implantação do software.

8. Ferramentas CASE

As ferramentas CASE (Computer-Aided Software Engineering) são softwares que ajudam no desenvolvimento e manutenção de sistemas, proporcionando suporte para análise, design e implementação.

9. Estruturação e Desenvolvimento de APIs

APIs (Application Programming Interfaces) são conjuntos de definições e protocolos que permitem a comunicação entre diferentes sistemas. O desenvolvimento de APIs consiste em definir os endpoints e a estrutura das respostas para facilitar a integração entre aplicações.

10. MVVM (Model View ViewModel)

O MVVM é um padrão de arquitetura de software que separa a lógica de interface (View), a lógica de apresentação (ViewModel) e a lógica de negócios (Model). Ele é usado principalmente em aplicações que requerem a atualização dinâmica de interfaces, facilitando o desenvolvimento e a manutenção do código.
