Engenharia de Requisitos

1. Conceitos Básicos

Engenharia de Requisitos é o processo de definir, documentar e gerenciar os requisitos de um sistema. Esses requisitos representam as necessidades e expectativas dos usuários e stakeholders em relação ao software.

2. Técnicas de Elicitação de Requisitos

Técnicas de elicitação de requisitos são métodos utilizados para coletar e identificar as necessidades dos usuários. Exemplos incluem entrevistas, workshops, questionários e análise de documentos.

3. Gerenciamento de Requisitos

Gerenciamento de requisitos envolve o rastreamento e controle das mudanças nos requisitos ao longo do ciclo de vida do projeto, garantindo que as necessidades dos stakeholders sejam atendidas.

4. Especificação de Requisitos

Especificação de requisitos é o processo de documentar os requisitos coletados de forma clara e compreensível, criando uma referência formal para o desenvolvimento do sistema.

5. Técnicas de Validação de Requisitos

Técnicas de validação de requisitos são usadas para garantir que os requisitos sejam precisos, completos e viáveis, evitando problemas futuros no desenvolvimento. Exemplos incluem revisões, inspeções e testes de protótipos.

6. Prototipação

Prototipação é a criação de versões preliminares do sistema para ajudar a entender e validar os requisitos. Os protótipos permitem que os usuários visualizem funcionalidades e forneçam feedback antes da implementação final.
