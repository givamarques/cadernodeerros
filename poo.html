Resumo de Análise e Projeto Orientados a Objetos e Interfaces de Interação com Usuário

1. Análise e Projeto Orientados a Objetos

1.1 Conceitos Básicos

A análise e projeto orientados a objetos são técnicas que utilizam conceitos como classes e objetos para modelar sistemas de software. Servem para criar sistemas mais estruturados e reutilizáveis, facilitando a manutenção e a evolução do software.

1.2 Padrões de Projeto

Padrões de projeto são soluções reutilizáveis para problemas recorrentes no desenvolvimento de software, promovendo boas práticas e facilitando a comunicação entre desenvolvedores.

1.3 UML: Visão Geral, Modelos e Diagramas

UML (Unified Modeling Language) é uma linguagem de modelagem utilizada para especificar, visualizar e documentar os elementos de um sistema de software. Inclui diversos diagramas, como diagrama de classes, diagrama de casos de uso, entre outros, para representar diferentes aspectos do sistema.

2. Programação Orientada a Objetos

2.1 Conceitos Básicos

  • Classes: Estruturas que definem propriedades e comportamentos dos objetos.
  • Objetos: Instâncias de classes que representam entidades do mundo real.
  • Métodos: Funções definidas dentro das classes que representam ações dos objetos.
  • Mensagens: Forma como os objetos se comunicam, chamando métodos de outros objetos.
  • Sobrecarga: Capacidade de definir múltiplos métodos com o mesmo nome, mas diferentes parâmetros.
  • Herança: Mecanismo que permite uma classe herdar características de outra classe.
  • Polimorfismo: Capacidade de um método ter diferentes comportamentos dependendo do contexto.
  • Interfaces: Contratos que definem um conjunto de métodos que devem ser implementados por uma classe.
  • Pacotes: Agrupamento de classes relacionadas, facilitando a organização e modularização do código.

2.2 Tratamento de Exceção

Tratamento de exceção é o mecanismo utilizado para lidar com erros e situações inesperadas em um programa, garantindo que ele continue funcionando de maneira segura.

3. Interface de Interação com Usuário

3.1 Conceitos de Interação Homem-Máquina

A interação homem-máquina (IHM) estuda como as pessoas interagem com sistemas computacionais, visando melhorar a eficiência e a experiência do usuário.

3.2 Interfaces Gráficas

Interfaces gráficas são componentes visuais de um sistema que permitem a interação do usuário por meio de janelas, botões, menus, etc.

3.3 Aspectos Cognitivos e Ergonômicos

Aspectos cognitivos e ergonômicos consideram a forma como o ser humano processa informações e interage fisicamente com os sistemas, garantindo que as interfaces sejam intuitivas e confortáveis.

3.4 Usabilidade e Avaliação

Usabilidade refere-se à facilidade de uso de um sistema, enquanto a avaliação de usabilidade é o processo de medir a eficiência, eficácia e satisfação dos usuários ao utilizar um sistema.

3.5 Interfaces Web

Interfaces web são as páginas e elementos com os quais o usuário interage em um navegador, incluindo botões, formulários e links.

3.6 Modelos, Métodos e Ferramentas

Os modelos, métodos e ferramentas usados no desenvolvimento de interfaces incluem técnicas como prototipagem, wireframes e testes de usabilidade, que ajudam a criar interfaces mais eficazes e alinhadas às necessidades dos usuários.
